## 💡 Key Usage Patterns

### Pattern 1: Code Generation
```
1. Open folder in VS Code (V1 or V2)
2. Claude Code automatically reads claude.md
3. Open CLAUDE_CODE_EXAMPLES.md → find relevant example
4. Copy prompt to Claude Code chat
5. Ask: "Generate [file] according to [phase] of my claude.md"
6. Review generated code
7. Copy to appropriate file
8. Test and commit
```

### Pattern 2: Debugging
```
1. Get error message in your code
2. Open Claude Code chat
3. Ask: "I'm getting this error: [error]. Based on my claude.md, what's wrong?"
4. Claude provides analysis and fix
5. Implement fix
6. Test and verify
7. Commit if working
```

### Pattern 3: Code Review
```
1. Have module/notebook created
2. Ask Claude: "Review [file] against my claude.md guidelines"
3. Claude checks for:
   - PEP 8 compliance
   - Documentation completeness
   - Type hints
   - Error handling
4. Make improvements
5. Commit
```

### Pattern 4: Optimization
```
1. Code works but not optimal
2. Ask Claude: "Optimize [code] for [speed/readability/memory]"
3. Get improvements with explanations
4. Implement
5. Test and verify
6. Commit
```
